4. Bảng điều khiển Giám sát

Chương 4

Bảng điều khiển Giám sát

Cung cấp cái nhìn trực quan, tập trung về hiệu suất cơ sở dữ liệu theo thời gian thực.

Mục tiêu

Hiển thị các chỉ số quan trọng, phát hiện xu hướng và bất thường.

Các công cụ

Amazon CloudWatch Dashboards

Tùy chỉnh các widget hiển thị biểu đồ từ:

  • CloudWatch Metrics
  • Log Groups (bao gồm Slow Query Logs)

Amazon RDS Performance Insights Dashboard

Cung cấp biểu đồ trực quan về:

  • Tải cơ sở dữ liệu
  • Phân tích các truy vấn SQL
  • Sự kiện chờ
  • Người dùng và máy chủ hàng đầu

Đây là công cụ chính để drill-down vào các vấn đề hiệu suất cấp độ DB.

Grafana (tùy chọn)

Nếu bạn muốn một dashboard tùy chỉnh hơn và tích hợp dữ liệu từ nhiều nguồn:

  • CloudWatch
  • Prometheus
  • Các nguồn dữ liệu khác

Nội dung hiển thị

  • Tổng quan sức khỏe DB: CPU, RAM, IOPS, Network
  • Tải DB: Average Active Sessions (AAS) và phân tích các wait event, SQL hàng đầu
  • Số lượng kết nối cơ sở dữ liệu
  • Tỷ lệ bộ nhớ đệm (buffer cache hit ratio)
  • Trạng thái Replication (nếu có Read Replicas)

4.1 Thiết lập CloudWatch Dashboard

Thiết lập CloudWatch Dashboard Bước 1: Truy cập CloudWatch Tìm kiếm “CloudWatch” trong AWS Console Click vào “Amazon CloudWatch” Bước 2: Tạo Dashboard Trong menu bên trái, click “Dashboards” Click “Create dashboard” Dashboard name: RDS-Performance-Dashboard Click “Create dashboard” Bước 3: Thêm Widget đầu tiên - CPU Utilization Chọn “Line” widget type Click “Next” Metrics: • Namespace: AWS/RDS • Metric name: CPUUtilization • DBInstanceIdentifier: Chọn instance vừa tạo Period: 5 minutes Click “Create widget” Bước 4: Thêm Widget thứ hai - Database Connections Click “Add widget” Chọn “Line” → “Next” Metrics: • Namespace: AWS/RDS • Metric name: DatabaseConnections • DBInstanceIdentifier: Chọn instance của bạn Click “Create widget” Bước 5: Thêm Widget thứ ba - Free Memory Click “Add widget” Chọn “Line” → “Next” Metrics: • Namespace: AWS/RDS • Metric name: FreeableMemory • DBInstanceIdentifier: Chọn instance của bạn Click “Create widget” Bước 6: Thêm Widget bổ sung Widget Read/Write IOPS Click “Add widget” Chọn “Line” → “Next” Thêm hai metrics: • ReadIOPS: Số lượng read operations per second • WriteIOPS: Số lượng write operations per second Click “Create widget” Widget Read/Write Latency Click “Add widget” Chọn “Line” → “Next” Thêm hai metrics: • ReadLatency: Độ trễ read operations • WriteLatency: Độ trễ write operations Click “Create widget” Widget Network Throughput Click “Add widget” Chọn “Line” → “Next” Thêm hai metrics: • NetworkReceiveThroughput: Lưu lượng mạng nhận • NetworkTransmitThroughput: Lưu lượng mạng gửi Click “Create widget” Bước 7: Tùy chỉnh Dashboard Sắp xếp Widgets Kéo thả các widgets để sắp xếp theo ý muốn Resize widgets bằng cách kéo góc Nhóm các metrics liên quan gần nhau Thiết lập Time Range Ở góc trên bên phải, chọn time range: • 1h - 15 phút Bước 8: Lưu và Chia sẻ Dashboard Lưu Dashboard Click “Save dashboard” Dashboard sẽ được lưu tự động Chia sẻ Dashboard Click “Share dashboard” Chọn permissions: • Public: Ai cũng có thể xem • Account: Chỉ account hiện tại • Specific users: Chỉ users được chỉ định Export Dashboard Click “Actions” → “View/edit source” Copy JSON configuration Có thể import vào account khác Bước 9: Best Practices Tổ chức Widgets Nhóm theo chức năng: • Performance metrics (CPU, Memory) • I/O metrics (IOPS, Latency) • Network metrics (Throughput) • Connection metrics (Connections, Queries)
